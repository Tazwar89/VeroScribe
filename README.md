# VeroScribe

For this exercise, we’d like you to build a simple patient booking flow to test your capabilities.

The feature should include:
- A patient-facing flow where a patient can choose a physician and request/book an appointment
- A way to select from available appointment times
- A basic form for patient details and reason for visit
- A physician/admin-facing view to see upcoming bookings
- Booking statuses such as pending, confirmed, or cancelled

You can use mock data, a local database, or any lightweight setup you think is appropriate. We are NOT expecting production-ready infrastructure, calendar integrations, payment flows, insurance logic, or full authentication.
